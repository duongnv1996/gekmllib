User Guide

Using gekmllib is, hopefully, straightfoward.

Requirements.
gekmllib requires certain jars to be on the classpath to operate correctly. These jars are packaged in the dependencies ZIP file on the website.

Usage
To load a KML document, you simply import com.keithpower.gekmllib.Kml and KMLParser, and then parse the document like so: 

Kml doc = new KMLParser().parse(new File("yourKMLfile.kml"));

You can access the root folder by calling doc.getFolder(). You can access the array of its folders with doc.getFolder().getFolders(). You can access a folders Placemarks, GroundOverlays etc with folder.getPlacemarks(), folder.getGroundOverlays() etc.

You can retrieve the KML corresponding to your document by calling: doc.toKML().

If you wish to produce KML Updates, for use over a NetworkLink, then calling doc.toUpdateKML() will return the KML Update which represents the changes that have happened to the document since toUpdateKML() was last called. Note: after loading a document, or first using it's KML, you will typically want to call toUpdateKML(), ignoring its output so that the nodes created by loading the document are not incorporated into the KML Update.

Parsing KML Fragments
It is possible to parse a fragment of a KML document. To do this you must initialise a KMLParser object with the name of the enclosing tag in your fragment, and the name of the class which represents this tag. For instance, to parse a GroundOverlay fragment you would do this:

KMLParser parser = new KMLParser("GroundOverlay", GroundOverlay.class);
GroundOverlay ge = (GroundOverlay)parser.parseFragment(
                                new File("groundoverlayfragment.kml"));

Examples
There are several examples in the pip folder, which show you how to load, create and modify KML documents and fragments.
